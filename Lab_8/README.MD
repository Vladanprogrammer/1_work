## Наступним кроком був створений мій власний агент, який працює за моєю ідеєю, та працюючий за рекомендаціями із методичного завдання, код: 
from google.adk.agents.llm_agent import Agent

def calculate_probability(m: int, n: int) -> dict:
    """
    Розраховує класичну ймовірність.
    
    Args:
        m: кількість сприятливих подій
        n: загальна кількість можливих подій
    """
    if n == 0:
        return {"error": "Ділення на нуль неможливе."}
    if m > n:
        return {"error": "Помилка: сприятливих подій не може бути більше за загальні."}
        
    return {"probability": m / n, "error": None}

root_agent = Agent(
    model='gemini-2.5-flash',
    name='root_agent',
    description='Ти розумний калькулятор який вираховує теорію ймовірності',
    instruction=(
        'Ти відповідаєш лише Українською мовою. '
        'Ти розумний калькулятор, який допомагає розрахувати теорію ймовірності. '
        'Обов\'язково використовуй інструмент calculate_probability для отримання результату. '
        'Ти видаєш лише чіткі відповіді на поставлену задачу з теорії ймовірності.'
    ),
    tools=[calculate_probability]
)

## Sequential Agent - Послідовне виконання![alt text](<Знімок екрана 2026-06-04 о 10.35.49.png>)![alt text](<Знімок екрана 2026-06-04 о 10.36.19.png>)
## Головна перевага Sequential агента — це побудова конвеєра, де складне завдання розбивається на прості етапи для вузькоспеціалізованих агентів Порядок виконання критично важливий, оскільки кожен наступний агент використовує вихідні дані (output) попереднього як свої вхідні дані (input). Наприклад, агент-аналітик фізично не зможе виконати свою роботу, якщо перед ним не відпрацює агент-пошуківець і не передасть йому зібрану інформацію. Зміна порядку просто зламає логіку ланцюжка
## Loop Agent - Циклічне виконання ![alt text](<Знімок екрана 2026-06-04 о 11.48.08.png>)Функція exit_loop працює як інструмент дострокового переривання циклу (аналог оператора break у програмуванні). Вона передається як інструмент (tool) агенту-оцінювачу. Після кожної ітерації агент аналізує проміжний результат, і якщо він повністю відповідає критеріям якості, ШІ самостійно викликає функцію exit_loop. Фреймворк фіксує цей виклик і миттєво зупиняє роботу циклу, не чекаючи вичерпання ліміту max_iterations, що економить час та обчислювальні ресурси
## Parallel Agent - Паралельне виконання![alt text](<Знімок екрана 2026-06-04 о 11.52.19.png>)Перевага паралельного виконання полягає в оптимізації часу за рахунок одночасної обробки незалежних задач. При послідовному виконанні загальний час дорівнює сумі часу кожного з трьох досліджень (T1+T2+T3). У паралельній архітектурі всі запити відправляються одночасно, тому загальний час очікування дорівнює лише часу виконання найдовшого з них (max(T1,T2,T3)). Це дозволяє ресурсам системи не простоювати в очікуванні відповіді та значно прискорює збір даних
